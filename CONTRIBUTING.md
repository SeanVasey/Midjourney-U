# Contributing to Midjourney-U

Thank you for your interest in contributing! Here's how to get started.

## Getting Started

1. Fork the repository
2. Create a feature branch from `main`: `git checkout -b feat/your-feature`
3. Make your changes
4. Ensure your changes follow the project standards (see `CLAUDE.md`)
5. Commit using [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`, `docs:`, etc.
6. Push to your fork and open a Pull Request

## Development

```bash
# Clone your fork
git clone https://github.com/<your-username>/Midjourney-U.git
cd Midjourney-U

# Create a branch
git checkout -b feat/your-feature
```

## Commit Messages

Use Conventional Commits format:

- `feat:` — New feature
- `fix:` — Bug fix
- `docs:` — Documentation changes
- `chore:` — Maintenance tasks
- `refactor:` — Code restructuring
- `test:` — Adding or updating tests

## Pull Requests

- Keep PRs focused — one concern per PR
- Update documentation if your change affects it
- All CI checks must pass before merge
- Bug fixes should include a regression test

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).
